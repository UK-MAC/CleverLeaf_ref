Detailed Description

Cleverleaf extends LagrangianEulerianPatchStrategy to provide necessary physics.

Cleverleaf implements the abstract methods in the LagrangianEulerianPatchStrategy class with the concrete versions of the methods needed to run the required physics on a patch.

Input Parameters

Definitions:

  • dim the dimension of the problem.
  • vis_dump_interval the frequency at which visualisation dumps are written.
  • field_summary_interval the frequency at which the field summary is calculated.
  • basename the basename to use for output files.
  • log_filename the filename for logging output.
  • log_all_nodes whether or not to log output from all nodes.
  • tag_all whether or not to tag all cells (used for debugging).
  • tag_q threshold for viscosity tagging.
  • tag_density gradient threshold for density tagging.
  • tag_energy gradient threshold for energy tagging.
  • states set of states.
  • num_states the number of states in the input deck.
  • state_n set of parameters describing state n.
  • geometry the geometry of the state. Can be either "RECTANGLE", "CIRCLE", or "POINT"
  • min lower-left coordinate of the state.
  • max upper-right coordinate of the state.
  • center center coordinate of the state (only for "CIRCLE" or "POINT").
  • radius radius of the state (only for "CIRCLE").
  • density initial density of the state.
  • energy initial energy of the state.
  • xvel initial x-velocity of the state.
  • yvel initial y-velocity of the state.

A sample input file entry might look like this:

    Cleverleaf {
        dim = 2
        vis_dump_interval = 1
        field_summary_interval = 5

        states {
            num_states = 2

            state0 {
                density = 0.2e0
                energy = 1.e0
            }

            state1 {
                geometry = "RECTANGLE"
                min = 0.e0, 0.e0
                max = 5.e0, 2.e0

                density = 1.e0
                energy = 2.5e0
            }
        }
    }
